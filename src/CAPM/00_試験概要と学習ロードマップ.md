# 試験概要と学習ロードマップ

## 1. この章で固めること

- 現行CAPM試験の構造と出題比率を把握する
- 受験要件（23 contact hours）の確認と講座選定
- 学習順と時間配分の戦略を決める
- 推奨リソースを選ぶ

---

## 2. CAPM試験の構造（2023年改訂後）

### 2.1 試験基本情報

| 項目 | 内容 |
|---|---|
| 問題数 | 150問（採点対象135問＋プレテスト15問） |
| 試験時間 | 180分 |
| 1問あたりの目安時間 | 72秒 |
| 形式 | 選択式（シナリオ型が中心） |
| 受験方法 | Pearson VUE 会場またはオンライン監督 |
| 日本語対応 | あり（試験中に日英切替可） |
| 受験要件 | 高卒以上 ＋ 23 contact hours のPM教育 |
| 合格ライン | PMI非公開（市場の目安：正解率60〜65%以上） |

### 2.2 出題ドメイン比率（ECO：Exam Content Outline 2023）

| ドメイン | 出題比率 | 主な出題内容 |
|---|---:|---|
| Project Management Fundamentals and Core Concepts | 36% | PM基礎、ライフサイクル、役割、計画、問題解決 |
| Predictive, Plan-Based Methodologies | 17% | WBS・CPM・EVM・品質・統合管理 |
| Agile Frameworks/Methodologies | 20% | Scrum・Kanban・反復計画・適応型の管理 |
| Business Analysis Frameworks | 27% | BAの役割、要求収集、トレーサビリティ、ロードマップ、受入基準 |

**重要**：2023年改訂後のCAPMは、PM基礎・予測型・アジャイル・ビジネス分析（BA）の4ドメインで構成される。
特にBAは27%を占めるため、要求収集・受入基準・トレーサビリティを後回しにすると合格確率が下がる。

### 2.3 準拠文書

| 文書 | 役割 |
|---|---|
| PMBOK 7th Edition | プロジェクト管理の基本原則と8パフォーマンスドメイン |
| PMBOK 6th Edition | 49プロセス・10知識エリアの定義（旧形式問題で参照される） |
| Agile Practice Guide | アジャイルとハイブリッドアプローチの基礎 |
| Business Analysis for Practitioners / PMI Guide to Business Analysis | BAの役割、要求、プロダクト価値の考え方 |

## 3. 受験要件の確認

### 3.1 23 contact hours の条件

- PMが認定する講座で23時間以上のPM教育を受けること
- 修了証（Certificate of Completion）が発行されること
- **「23 contact hours」が明記されていない講座は要件を満たさない**

### 3.2 講座選定で確認すること

| チェック項目 | 確認ポイント |
|---|---|
| 修了証の発行 | 必ず修了証が出るか |
| 時間数の明記 | "23 contact hours" または "23 PDUs" の記載があるか |
| 試験範囲の網羅 | PMBOK 7th ＋ Agile Practice Guide を含むか |
| 価格と期間 | 費用対効果と自分のペースに合うか |

### 3.3 受験申請の流れ

1. PMI公式サイトでアカウント作成
2. 23 contact hours の教育実績を入力
3. 申請書を提出（審査期間：数日〜1週間）
4. 承認後、Pearson VUEで試験日を予約
5. 試験（会場またはオンライン）

## 4. 学習戦略

### 4.1 推奨学習順

| 順番 | ファイル | 習得目標 | 目安時間 |
|---|---|---|---|
| 1 | この章（`00_`） | 試験構造と弱点領域を把握する | 1時間 |
| 2 | `01_PM基礎とコア概念.md` | PMBOK体系とPM基礎を固める | 3時間 |
| 3 | `02_予測型計画とコントロール.md` | WBS・CPM・EVMを計算できる水準にする | 5時間 |
| 4 | `03_リスク_品質_変更管理.md` | PMI定義の対応戦略と品質ツールを覚える | 3時間 |
| 5 | `04_アジャイルとハイブリッド.md` | Scrum・Kanban・ハイブリッド判断を固める | 3時間 |
| 6 | `05_ビジネス分析.md` | BAの役割・要求収集・受入基準を覚える | 3時間 |
| 7 | `06_ステークホルダーとコミュニケーション.md` | 分析グリッドと公式を覚える | 2時間 |
| 8 | `07_実務ケースで覚えるCAPM.md` | 知識を案件シナリオに結びつける | 2時間 |
| 9 | `08_問題演習とひっかけ辞典.md` | 実戦形式で弱点を発見し潰す | 3時間 |
| 10 | `09_試験直前チェックリスト.md` | 公式・定義の暗記漏れを洗う | 1時間 |

合計目安：**26〜35時間**（基礎知識がある場合は短縮可能）

### 4.2 時間配分戦略（試験当日）

| 局面 | 戦略 |
|---|---|
| 1問あたりの上限 | 72秒を超えたら即フラグして次へ |
| フラグ問題の処理 | 全問通過後に残り時間で戻る |
| 計算問題 | ホワイトボードを積極的に使う。数値を書いてから計算する |
| シナリオ型 | 「PMIはPMをどう行動させたいか」の視点で選ぶ |
| 迷ったときの原則 | 早め・先手・プロセス通りの選択肢を選ぶ |

### 4.3 PMIが好む正解パターン

CAPMはPMIの価値観に基づいた試験である。以下のパターンを覚えておく。

| 状況 | PMIが好む答え | PMIが嫌う答え |
|---|---|---|
| 問題が起きそう | 先に特定してリスク対応する | 起きてから対処する |
| 変更要求が来た | 変更管理プロセスに従う | 即受諾または即拒否する |
| 遅延の予兆がある | 早期に報告・選択肢を出す | 確定するまで様子を見る |
| メンバーが困っている | コーチングして自己解決を促す | 代わりにやってあげる |
| ステークホルダーが不満 | 期待値管理・説明する | 無視する・後で説明する |

## 5. 推奨学習リソース

### 5.1 公式リソース

| リソース | 内容 | 入手方法 |
|---|---|---|
| PMBOK 7th Edition | PMIの公式ガイド | PMI会員（無料）またはPMIストアで購入 |
| Agile Practice Guide | アジャイル公式ガイド | PMIサイトで無料DL可 |
| Business Analysis for Practitioners | BAドメインの要求・受入基準・検証の基礎 | PMIサイト/PMIストア |
| ECO（Exam Content Outline） | 出題範囲の公式定義 | PMIサイトで無料DL可 |

### 5.2 参考書（第三者リソース）

| 書籍/講座 | 特徴 | 適した使い方 |
|---|---|---|
| Joseph Phillips の CAPM講座（Udemy） | 包括的・英語・コスパ高い | メイン学習に使える |
| Andrew Ramdayal の PMI試験対策（TIA） | シナリオ型問題に強い | 問題演習に向く |
| Rita Mulcahy の PM Exam Prep | PMP向けだが内容が深い | 概念を深掘りしたいとき |

### 5.3 模擬問題

| ツール | 特徴 |
|---|---|
| PMI公式サンプル問題 | 少ないが出題傾向を把握できる |
| PrepCast（有料） | 本番に近い難易度のシミュレーター |
| Exam Topics / Quizlet | 無料で使えるが質はばらつきがある |

**推奨：模擬試験で70%以上を安定して取れるまで繰り返す。**

## 6. よくある失敗パターン

| 失敗パターン | なぜ失敗するか |
|---|---|
| BAを後回しにする | 27%の大きなドメインで失点しやすい |
| アジャイルを後回しにする | 20%のドメインに加え、ハイブリッド判断でも失点しやすい |
| EVM計算を「なんとなく理解」で済ます | 計算問題で確実に得点を落とす |
| 実務経験だけで乗り越えようとする | PMIの定義と実務は一致しないことがある |
| 23 contact hoursを雑に扱う | 修了証が出ない講座で詰まる |
| 暗記だけで乗り越えようとする | シナリオ型は知識の応用を問う |

---

## 7. 最後に覚える一文

CAPM は「実務センス」ではなく「PMIの定義に従ってプロジェクトを管理する知識」を問う試験である。
PMIが期待する行動パターンを問題の文脈から読む力が、合格の核心である。
