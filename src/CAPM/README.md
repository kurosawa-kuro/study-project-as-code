# CAPM 参考書ガイド

このディレクトリは、CAPM を「取る価値はあるが、PM 専業化までは寄せない」という前提で学ぶための参考書本体である。
目的は、暗記で受かることではなく、リモート技術リードとして必要な PM 管理観点を言葉にできる状態を作ることにある。

## この教材の使い方

1. まず [`01_受験要件と学習戦略.md`](./01_受験要件と学習戦略.md) で、なぜ取るのかと、どこまで学ぶのかを固定する
2. 次に [`02_スコープ_スケジュール_コスト.md`](./02_スコープ_スケジュール_コスト.md) で、計画の土台となる 3 観点を固める
3. 続いて [`03_リスク_品質_変更管理.md`](./03_リスク_品質_変更管理.md) で、炎上を未然に切る観点を学ぶ
4. その後 [`04_ステークホルダー_コミュニケーション.md`](./04_ステークホルダー_コミュニケーション.md) で、リモート推進に直結する説明力へ落とす
5. 次に [`05_実務ケースで覚えるCAPM.md`](./05_実務ケースで覚えるCAPM.md) で概念を案件へ戻す
6. 続いて [`06_試験直前チェックリスト.md`](./06_試験直前チェックリスト.md) で説明力の穴を洗う
7. 最後に [`07_ひっかけ辞典と章末問題.md`](./07_ひっかけ辞典と章末問題.md) で概念差と誤答ポイントを確認する

## 読みながら意識すること

- 用語を覚える前に、「その概念がないと何が壊れるか」を考える
- PM の一般論として読むのではなく、今の案件でどう説明するかに引き戻す
- 各章の「ひっかけパターン」を、自分が普段やってしまいがちな行動に置き換える
- 受験勉強と実務説明力を分離しない

## この教材の到達点

- CAPM をなぜ取るのかを 30 秒で説明できる
- スコープ、スケジュール、コスト、リスク、品質、変更、ステークホルダー、コミュニケーションを自分の案件で話せる
- 追加要望、遅延、認識ズレが起きたときに、どの観点で整理すべきか即答できる
- 試験前に暗記の穴ではなく、判断の穴を埋める学習ができる
