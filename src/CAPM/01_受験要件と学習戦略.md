# 受験要件と学習戦略

## 1. この章で切れる選択肢

- 受験要件を後回しにして、先に問題演習へ入る
- 23 時間教育要件を先に満たしてから学習に入る
- PM 専業レベルまで深掘りしてから受ける
- 合格点突破型で、実務に返せる範囲に絞って進める

## 2. 技術の最短定義

CAPM は、プロジェクト管理の基礎知識を体系的に問う知識試験である。
この資格の価値は、管理職転向の宣言ではなく、「技術リードとして管理観点の会話ができる」と第三者に示しやすくなる点にある。

ここで大事なのは、CAPM を偉い資格として扱うことではない。
今の自分に足りないのが「価値判断の語彙」ではなく、「工数、スコープ、リスク、期待値調整をどう説明するか」なら、CAPM は投資対象になる。

## 3. 今の状態で選ぶ条件

- 実装や技術判断はできるが、予算、人月、スコープの会話になると説明が弱くなる
- リモート環境で、報告や論点整理の再現性をもっと強くしたい
- 次案件で「ただの実装者」ではなく「任せられる技術リード」として見せたい
- PM 専業に転身したいわけではないが、最低限の PM 管理観点は持っておきたい

## 4. 今の状態で選ばない条件

- 資格を増やすこと自体が目的化している
- CAPM を取ればマネージャーっぽく見える、という見え方だけを狙っている
- PMLE や技術学習を止めるほど重い計画になっている
- 受験要件や講座条件を確認せず、「たぶん後で何とかなる」と進めようとしている

## 5. ひっかけパターン

- 試験勉強を始めた瞬間に、急に PM 専業向けの参考書や深い理論書まで買い足す
- 23 時間教育要件を雑に扱い、修了証が出ない講座を受けて後から詰まる
- 単語カードだけで学び、「その概念が現場で何を防ぐのか」を結びつけない
- PMP と同じテンションで学び始め、時間だけ溶かす

## 6. 代表パターン

### 6.1 まず押さえる試験前提

| 項目 | 内容 |
|---|---|
| 実技 | なし |
| 問題数 | 150 問 |
| 試験時間 | 180 分 |
| 形式 | 選択式中心 |
| 受験方法 | Pearson VUE 会場またはオンライン監督 |
| 日本語対応 | あり |
| 受験要件 | 高卒相当以上 + 23 時間以上の PM 教育 |

### 6.2 この資格で本当に補いたいもの

| 足りていないもの | CAPM で補えること |
|---|---|
| 工数や人月の会話が弱い | コストと計画の基礎語彙 |
| スコープ変更の説明が弱い | 変更管理と影響整理の語彙 |
| リスク報告が感覚的 | リスク識別と対応の枠組み |
| 報告はできるが整理が弱い | ステークホルダーとコミュニケーション設計 |

### 6.3 学習順の型

1. 受験要件を確認する
2. 23 時間教育を満たす講座を確定する
3. 主要観点を章ごとに理解する
4. 模擬問題で弱点観点を洗う
5. 実務の言い換えを作る
6. 受験日を決める

## 7. 実務へのつなぎ方

### 7.1 「取る理由」の言い方

悪い言い方:

```text
管理もできるようになりたいので CAPM を取ります。
```

弱い。抽象的で、技術主軸との接続が見えない。

良い言い方:

```text
技術判断だけでなく、工数・スコープ・リスク・期待値調整を
リモートでも再現性高く回せる技術リードであることを示すために CAPM を取る。
```

### 7.2 講座選定で見るポイント

- 修了証が必ず出るか
- `23 contact hours` が明記されているか
- 試験対策として最低限まとまっているか
- 深すぎて本筋を逸れないか

### 7.3 勉強中に自分へ問い続けること

- これは試験のための暗記か、現場の説明力につながる理解か
- この概念がないと、どの会話で困るか
- 今の案件でこの観点を 1 分で説明できるか

## 8. 最後に覚える一文

CAPM は、技術リードが「作れる」だけでなく「管理観点でも会話できる」と示すための基礎証明である。
