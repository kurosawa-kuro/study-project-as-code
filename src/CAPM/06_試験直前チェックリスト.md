# 試験直前チェックリスト

## 1. 受験前に確認すること

- 23 時間教育要件の修了証を保存している
- 試験形式、問題数、試験時間、受験方法を把握している
- 受験日を決めた理由が曖昧でない
- PMLE や実務を圧迫する無理な詰め込みになっていない

## 2. 理解が浅いと危ない観点

- スコープと要件追加の違い
- 遅延とリスクの違い
- 品質とテストの違い
- 変更管理と単なる拒否の違い
- ステークホルダー管理と全員満足の違い
- コミュニケーション管理と報告量の違い

## 3. 実務に言い換えられるか確認する質問

- 追加要望が来たとき、最初に何を見るか
- 遅延の予兆が出たとき、誰に何を伝えるか
- 品質をどう定義してからレビューに入るか
- 変更要求をどう整理して返すか
- 顧客、上位、メンバーに同じ資料を出さない理由は何か

## 4. 30 秒で言えるようにしたい説明

### なぜ CAPM を取るのか

```text
技術判断だけでなく、工数・スコープ・リスク・期待値調整を
リモートでも再現性高く回せる技術リードであることを示すため。
```

### なぜ PSPO I ではなく CAPM が本命なのか

```text
価値判断の語彙は PSPO I 学習で回収できるが、
予算・人月・スコープ・リスク管理の外部証明は CAPM の方が直撃するため。
```

## 5. 最後の確認

- 用語暗記だけで終わっていないか
- 自分の案件で 1 つでも具体例を言えるか
- 「何を守るための概念か」を説明できるか

## 最後に覚える一文

試験直前に見るべきなのは単語の数ではなく、自分がどの判断観点をまだ実務で言い換えられないかである。
