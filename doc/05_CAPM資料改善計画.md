# CAPM 資料改善計画

## 1. 現状分析

### 1.1 改善に至った判断

`src/CAPM/` の既存7ファイルは「実務で使える技術リード思考」を鍛える読み物として一定の品質を持つが、
CAPM試験合格という目的に対しては根本的に不足している。

現状の資料が合格につながらない理由は以下の3点に集約される。

| 問題 | 具体的な欠落 | 試験への影響 |
|---|---|---|
| 計算問題への対応がゼロ | EVM公式・CPM計算が一切未記載 | EVM問題だけで試験の10〜15%を占める |
| PMBOK知識体系が未整理 | プロセスグループ・知識エリアの体系がない | 出題の骨格そのものが欠落 |
| アジャイル知識が皆無 | Scrum・Kanban・ハイブリッドの記述なし | 現行CAPM試験の約50%がアジャイル問題 |
| 演習問題が極端に少ない | 5問・単純選択肢のみ | 試験は150問・シナリオ型 |
| PMI固有定義が不正確 | リスク対応戦略4種など定義語が未記載 | 定義の取り違えで確実に失点する |

### 1.2 現行CAPM試験の構造（2023年改訂後）

| 項目 | 内容 |
|---|---|
| 問題数 | 150問（採点対象135問＋プレテスト15問） |
| 試験時間 | 180分（1問あたり72秒が目安） |
| 出題比率 | Predictive（ウォーターフォール）約50%、Agile/Hybrid約50% |
| 準拠文書 | PMBOK 7th edition ＋ Agile Practice Guide |
| 合格ライン | PMIは非公開だが市場では60〜65%以上が目安 |
| 受験要件 | 高卒以上＋23 contact hours（PMの教育） |

## 2. 改善方針

### 2.1 基本方針

既存7ファイルの「実務思考フレーム」としての価値は保持する。
ただし現状は「目次に毛が生えた程度のメモ」であるため、
「試験で問われる具体知識」を各章に追加することで合格水準に引き上げる。

追加する軸は2つである。

1. **知識の具体化**：公式名・ツール名・定義語をPMI準拠で明記する
2. **演習の強化**：計算問題を含む20問以上の実戦形式問題を整備する

### 2.2 改善しない範囲

- 既存の実務ケース（`05_実務ケースで覚えるCAPM.md`）の構成は維持する
- 哲学的な「なぜこの概念が必要か」の説明は削除しない
- ファイル名・順番の大枠は変えない（`08_` の新規追加のみ）
