# study-leader-skill

リモート技術リードとしての信用補強に必要な資格戦略を整理するためのドキュメントリポジトリ。
現時点ではアプリケーション実装が主目的ではなく、資格方針・ロードマップ・運用ルールを明文化し、判断のぶれを減らすことを目的とする。

## 現在の位置づけ

- 主対象: リーダースキル資格の仕様整理
- 主眼: PM 専業化ではなく、技術主軸を保ったまま外部証明を補強すること
- 現在の結論: `PMLE -> PSPO I 学習のみ -> CAPM 取得 -> Databricks 基礎資格`

## リポジトリ構成

```text
.
├── AGENTS.md        # AI コーディングエージェント共通ガイド
├── CLAUDE.md        # Claude Code 向け補助ガイド
├── Makefile         # ひな形コマンド
├── env/
│   ├── config.yaml  # 非機密設定
│   └── secret.yaml  # ローカル秘密情報（コミット禁止）
└── doc/
    ├── 01_仕様と設計.md
    ├── 02_移行ロードマップ.md
    ├── 03_実装カタログ.md
    ├── 04_運用.md
    └── README.md
```

## ドキュメント

ドキュメントの権威順位と更新規約は [`doc/README.md`](doc/README.md) に従う。

- [`doc/01_仕様と設計.md`](doc/01_仕様と設計.md) — 資格戦略の仕様と設計の整理
- [`doc/02_移行ロードマップ.md`](doc/02_移行ロードマップ.md) — 何を採用し、何を保留するかの決定的仕様
- [`doc/03_実装カタログ.md`](doc/03_実装カタログ.md) — リポジトリ構成や実装物のカタログ
- [`doc/04_運用.md`](doc/04_運用.md) — 更新運用、環境構築、定常手順

## 運用メモ

- このリポジトリはまだテンプレート要素を含むため、内容の具体化時は関連ドキュメントを同一変更で更新する
- 非機密は `env/config.yaml`、ローカル秘密情報は `env/secret.yaml`、共有・本番秘密情報は Doppler（`doppler.yaml`）で管理する
- `env/secret.yaml` はコミットしない
