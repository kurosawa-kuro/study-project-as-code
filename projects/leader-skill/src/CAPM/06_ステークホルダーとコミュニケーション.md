## 2. ステークホルダー管理

### 2.1 ステークホルダーの定義

ステークホルダーとは、プロジェクトの成果に影響を与える、または影響を受ける個人・グループ・組織のことである。

**重要**：ステークホルダーの特定は立上げプロセス（Initiating）から始める。プロジェクト開始後にも随時追加される。

### 2.2 ステークホルダーエンゲージメントレベル

| レベル | 意味 | PMの対応 |
|---|---|---|
| 無関心（Unaware） | プロジェクトを知らない | 情報提供して認知させる |
| 抵抗（Resistant） | 変化に反対している | 懸念を理解し対話する |
| 中立（Neutral） | 支持も反対もしない | メリットを説明する |
| 支持（Supportive） | 変化を受け入れている | 維持・活用する |
| 先導（Leading） | 積極的に変化を推進している | 協力を続ける |

**目標**：ステークホルダーを「無関心」「抵抗」から「支持」「先導」へ動かすこと。

### 2.3 ステークホルダー分析の4グリッド（必ず覚える）

#### 権力/関心マトリクス（Power/Interest Grid）

最も頻出。「権力（意思決定力）」と「関心（プロジェクトへの興味）」で4象限に分類する。

| 象限 | 戦略 |
|---|---|
| 権力：高、関心：高 | 緊密に管理する（Manage Closely） |
| 権力：高、関心：低 | 満足させる（Keep Satisfied） |
| 権力：低、関心：高 | 情報を提供し続ける（Keep Informed） |
| 権力：低、関心：低 | 最低限の監視を続ける（Monitor） |

#### 権力/影響マトリクス（Power/Influence Grid）

同様の構造。「影響」はプロジェクトへの積極的な参与度を指す。

#### 影響/インパクトグリッド（Influence/Impact Grid）

影響力（Influence）と、変化を受けるインパクト（Impact）で分類する。

#### サリエンスモデル（Salience Model）

3つの属性で関心度を測る。

| 属性 | 意味 |
|---|---|
| 権力（Power） | 自分の意志を押し通せる力 |
| 正当性（Legitimacy） | 関与することの正当な理由があるか |
| 緊急性（Urgency） | 即座の対応を要する要求があるか |

3属性を全て持つステークホルダー → 最優先で対応する。

## 3. コミュニケーション管理

### 3.1 コミュニケーションチャンネル数の公式（必ず覚える）

プロジェクトメンバーが N 人の場合：

```
チャンネル数 = N × (N - 1) / 2
```

| メンバー数 | チャンネル数 |
|---|---|
| 2人 | 1 |
| 5人 | 10 |
| 10人 | 45 |
| 15人 | 105 |

**重要**：人数が増えるとチャンネル数は急増する。これがコミュニケーション管理が重要な理由である。

**よく出る計算問題の形式**：
「メンバーが5人から10人に増えた。コミュニケーションチャンネルはいくつ増えたか？」
→ 45 - 10 = **35本増加**

### 3.2 コミュニケーション手段の3分類（必ず覚える）

| 種類 | 英語 | 意味 | 例 |
|---|---|---|---|
| インタラクティブ | Interactive | 双方向・リアルタイムでやり取りする | 会議、電話、ビデオ会議 |
| プッシュ | Push | 送信者が受信者に一方向で送る | メール、報告書、プレスリリース |
| プル | Pull | 受信者が必要に応じてアクセスする | ポータルサイト、Wikiページ、イントラネット |

**ポイント**：緊急・重要な情報はインタラクティブが最も確実。大量・一斉周知はプッシュ。自己学習型はプル。

### 3.3 コミュニケーション計画書の構成要素

| 要素 | 内容 |
|---|---|
| ステークホルダーの情報ニーズ | 誰が何の情報を必要とするか |
| 情報の種類 | 進捗報告・リスク報告・変更通知 等 |
| 配布方法 | メール・会議・ポータル 等 |
| 頻度 | 毎日・週次・月次 等 |
| 担当者 | 誰が情報を作成・配布するか |
| エスカレーション経路 | 問題が解決しない場合の連絡先 |

### 3.4 コミュニケーションの障壁

| 障壁 | 内容 |
|---|---|
| 言語・文化的差異 | 用語の解釈が異なる |
| 情報過多 | 情報が多すぎて本質が埋もれる |
| 受け取り側のフィルタリング | 聞きたいことだけ聞く |
| 信頼関係の欠如 | 情報を共有しない・隠す |
| 地理的分散 | リモート・タイムゾーン差による遅れ |

### 3.5 相手ごとに必要な情報の粒度

| 相手 | 主に気にすること | 必要な伝え方 |
|---|---|---|
| スポンサー/顧客 | 影響、優先順位、意思決定 | 選択肢と影響を簡潔に |
| 上位管理職 | 安定推進、リスク、報告品質 | 論点と見通しを整理して |
| 開発メンバー | 実行条件、依存関係、役割 | 具体的で行動可能な形で |

## 4. ひっかけパターン

| ひっかけ | 正しい理解 |
|---|---|
| ステークホルダーの特定は計画段階から | 立上げ（Initiating）から始める |
| メンバーを増やすとコミュニケーションが楽になる | チャンネル数は急増。複雑になる |
| プッシュ型はいつでも最適 | 緊急・重要な事項はインタラクティブが最適 |
| 抵抗するステークホルダーは無視する | 抵抗を放置するとリスクになる。対話して理解を得る |
| コミュニケーション計画は一度作れば完成 | プロジェクト中に随時見直す必要がある |
| 情報量が多いほど良いコミュニケーション | 相手が判断できる形に整えることが目的 |

---

## 5. 最後に覚える一文

コミュニケーションチャンネル数は N(N-1)/2。
ステークホルダーは「権力×関心」マトリクスで優先度を決め、
「誰に・何を・いつ・どの手段で」を計画として明文化することがコミュニケーション管理の核心である。
