## 2. スコープ管理

### 2.1 スコープの定義

スコープは「今回どこまで責任を持つか」の境界である。

| 用語 | 定義 |
|---|---|
| プロダクトスコープ | 製品・サービスが持つべき機能や特性 |
| プロジェクトスコープ | プロダクトスコープを実現するために必要な作業 |
| スコープクリープ | 承認なしにスコープが少しずつ拡大すること |

### 2.2 WBS（Work Breakdown Structure）

WBSはプロジェクトの成果物と作業を階層的に分解した図である。

**100%ルール**：WBSの各レベルの合計が、上位レベルの100%を表すこと。余分なものは含まない。

| 概念 | 内容 |
|---|---|
| ワークパッケージ | WBSの最下位要素。実際に作業を割り当てる単位 |
| WBS辞書 | 各WBS要素の詳細説明（スコープ、完了基準、担当者、期間等）を記録した文書 |
| スコープベースライン | 承認されたスコープ記述書＋WBS＋WBS辞書の3点セット |

**WBSを作るときの原則**：
- 成果物ベースで分解する（活動ベースではない）
- 最下位要素（ワークパッケージ）で見積・監視ができる粒度にする
- 外部から調達するものもWBSに含める

## 3. スケジュール管理

### 3.1 アクティビティの依存関係（PDM）

PDM（Precedence Diagramming Method）はアクティビティをノードで表すネットワーク図である。

| 依存関係 | 略称 | 意味 | 例 |
|---|---|---|---|
| Finish-to-Start | FS | Aが終わってからBが始まる | 設計が終わってから開発開始 |
| Finish-to-Finish | FF | Aが終わってからBが終わる | テストが終わってから報告書も終わる |
| Start-to-Start | SS | Aが始まってからBが始まる | 基礎工事が始まってから配管工事開始 |
| Start-to-Finish | SF | Aが始まってからBが終わる | 最も稀。後継が始まれば前任は終わる |

### 3.2 クリティカルパス法（CPM）

**クリティカルパス**：最長のパス経路。プロジェクトの最短完了日を決定する。

**フロート（余裕時間）**：

| 種類 | 計算式 | 意味 |
|---|---|---|
| 総フロート（Total Float） | LS - ES または LF - EF | スケジュール全体を遅らせずに遅らせられる最大日数 |
| 自由フロート（Free Float） | 後続のES - 自分のEF | 後続アクティビティを遅らせずに遅らせられる最大日数 |

**用語の略称**：
- ES（Early Start）：最も早く開始できる日
- EF（Early Finish）：最も早く終了できる日
- LS（Late Start）：全体遅延なしに最も遅く開始できる日
- LF（Late Finish）：全体遅延なしに最も遅く終了できる日

**クリティカルパス上のアクティビティは総フロートが0（ゼロ）である。**

#### CPM計算の例

```
A(3日) → C(4日) → E(2日)
A(3日) → D(2日) → E(2日)
B(5日) → D(2日) → E(2日)
```

- パス1: A→C→E = 3+4+2 = **9日**
- パス2: A→D→E = 3+2+2 = **7日**
- パス3: B→D→E = 5+2+2 = **9日**

クリティカルパスはパス1とパス3（各9日）。プロジェクト期間は**9日**。

### 3.3 スケジュール短縮の2手法

| 手法 | 方法 | リスク | コスト |
|---|---|---|---|
| ファストトラッキング | 本来順次のアクティビティを並行して実施する | 手戻りのリスクが高まる | 増加しない |
| クラッシング | 追加リソースを投入してアクティビティを短縮する | 低い | 増加する |

**ポイント**：ファストトラッキングはリスクを増加させる。クラッシングはコストを増加させる。

## 4. コスト管理とEVM

### 4.1 EVM（Earned Value Management）の基本概念

EVMはプロジェクトの進捗と予算消化を客観的に測定する手法である。

| 指標 | 英語名 | 定義 | 覚え方 |
|---|---|---|---|
| BAC | Budget at Completion | プロジェクト全体の計画予算 | 総予算 |
| PV | Planned Value | この時点までに完了しているはずの作業の計画コスト | 計画値 |
| EV | Earned Value | この時点で実際に完了した作業の計画コスト | 実績値（計画ベース） |
| AC | Actual Cost | この時点で実際に使ったコスト | 実際のコスト |

### 4.2 EVM公式一覧（必ず覚える）

#### 差異分析（マイナスが悪い）

| 公式 | 意味 | マイナスの解釈 |
|---|---|---|
| CV = EV - AC | コスト差異（Cost Variance） | コスト超過 |
| SV = EV - PV | スケジュール差異（Schedule Variance） | 遅延 |

#### 効率指数（1.0未満が悪い）

| 公式 | 意味 | 1.0未満の解釈 |
|---|---|---|
| CPI = EV / AC | コスト効率指数（Cost Performance Index） | 予算に対してコスト超過 |
| SPI = EV / PV | スケジュール効率指数（Schedule Performance Index） | スケジュール遅延 |

#### 予測値

| 公式 | 意味 | 使う場面 |
|---|---|---|
| EAC = BAC / CPI | 完成時総コスト予測（最一般的） | 今後も現在の効率が続く場合 |
| EAC = AC + (BAC - EV) | 完成時総コスト予測（残作業は計画通り） | 残作業は当初の計画どおりに進む場合 |
| ETC = EAC - AC | 完成のために必要な残コスト予測 | あといくらかかるかを知りたいとき |
| VAC = BAC - EAC | 完成時差異（最終的な予算超過/余剰） | プラスなら予算内、マイナスなら超過 |

#### 効率目標

| 公式 | 意味 |
|---|---|
| TCPI = (BAC - EV) / (BAC - AC) | 残予算内で完成するために必要な今後のコスト効率 |

**TCPI > 1.0**：現在より高い効率が必要（達成困難）
**TCPI < 1.0**：現在より低い効率でも完成できる（余裕あり）

### 4.3 EVM計算演習

**問題1**
プロジェクトのBAC = 100万円。現時点でPV = 60万円、EV = 50万円、AC = 55万円。
CV、SV、CPI、SPIをそれぞれ求めよ。

> **解答**
> - CV = EV - AC = 50 - 55 = **-5万円**（コスト超過）
> - SV = EV - PV = 50 - 60 = **-10万円**（スケジュール遅延）
> - CPI = EV / AC = 50 / 55 = **0.91**（コスト超過傾向）
> - SPI = EV / PV = 50 / 60 = **0.83**（スケジュール遅延傾向）

**問題2**
上記の状況でEACとETCを求めよ（今後も現在の効率が続く前提）。

> **解答**
> - EAC = BAC / CPI = 100 / 0.91 = **約110万円**
> - ETC = EAC - AC = 110 - 55 = **約55万円**

**問題3**
BAC = 200万円、EV = 80万円、AC = 100万円。TCPIを求めよ。

> **解答**
> - TCPI = (BAC - EV) / (BAC - AC) = (200 - 80) / (200 - 100) = 120 / 100 = **1.20**
> - 今後は現在より20%高い効率が必要。達成困難な水準。

**問題4**
プロジェクトのCPI = 0.8、BAC = 500万円。EACを求めよ。

> **解答**
> - EAC = BAC / CPI = 500 / 0.8 = **625万円**
> - 25%のコスト超過が見込まれる。

**問題5**
SPI = 1.2の意味として正しいのはどれか。
A. スケジュールが遅延している  B. スケジュールが20%前倒しになっている  C. コストが20%超過している  D. コストが節約できている

> **解答：B**
> SPI = EV/PV = 1.2 は「計画した量より20%多く完了している」ことを意味する。スケジュール前倒し。

## 5. ひっかけパターン

| ひっかけ | 正しい理解 |
|---|---|
| EV = 実際に使ったコスト | EV = 完了した作業の**計画**コスト（ACではない） |
| CPI < 1.0 = スケジュール遅延 | CPI はコストの指標。スケジュールはSPI で見る |
| クリティカルパス上のフロートは常にゼロ | 正しい。クリティカルパス上の総フロートは0 |
| ファストトラッキングはコストを増加させる | コスト増加はクラッシング。ファストトラッキングはリスク増加 |
| WBSは活動ベースで作る | WBSは**成果物**ベースで作る |

---

## 6. 最後に覚える一文

EVM は「プロジェクトが計画どおりに進んでいるか」を数値で示す唯一の客観指標である。
CPI と SPI が1.0 未満なら問題がある。EAC は「今後も同じ効率なら最終的にいくらかかるか」を示す。
