# ハルシネーションチェック

結論や計画を書く前に、主張を実際のリポジトリで検証する。

## 手順

1. `rg --files` でファイルとディレクトリの存在を確認する。
2. `rg` でシンボル・関数・ルート・テーブル・config キー・Makefile target・コマンド・テスト名・ドキュメント参照を確認する。
3. 重要な主張ごとにファイルパスを引用 / 明示する。
4. 見つからないものは推測せず「未確認」と記す。
5. 主張が実行時挙動に依存する場合、検証に必要なコマンドやテストを特定する。

## 出力

- パス付きの確定事実
- 未確認 / 根拠不足
- 実行したコマンド
- 追加で必要な検証
