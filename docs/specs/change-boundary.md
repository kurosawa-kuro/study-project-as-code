# Change Boundary（このプロジェクトの instantiation）

**Layer 5: Change Boundary**（§13）の repo 固有 instantiation。エージェントがコードを変更してよい場所と、owner 判断のため止まるべき場所を*このリポジトリ*向けに定義する。Skill: `control-change`。ファイル編集のハード強制は `detect-safety-boundary` hook（`.claude/hooks/`）。

> **使い方（厚いメニュー → 削る）**: 下記は「個人プロジェクトでよく保護するパス」を厚く列挙したもの。**このプロジェクトに存在しない/該当しないパスは削る**。`hooks/detect-safety-boundary.sh` の `case` と必ず一致させる（メニューを削ったら hook も削る）。

## 保護パス（owner approval が必要 / hook が exit 2 で止める）

```text
env/secret/**            # materialized secret files（gitignore 済みだが owner-sensitive）
infra/**                 # デプロイ基盤
**/terraform/**          # IaC（state を動かす）
**/pulumi/**             # IaC（別系）
**/migrations/**         # DB スキーマ migration
.github/workflows/**     # CI/CD
**/k8s/**, **/helm/**    # クラスタ構成
**/*.tfvars, **/*.tfstate   # IaC 変数・状態
Dockerfile, **/docker-compose*.yml   # ランタイムイメージ
```

> 該当しない行は削る。逆に、このプロジェクト固有の保護対象（例: 本番/マスターデータ置き場、生成設定ファイル）があれば足す。

## owner 判断領域（hook では止まらないが Heavy / `control-change` 対象）

- 本番 / マスターデータ（このプロジェクトの loss-critical 資産 — `capability-boundary.md` で定義したもの）
- DB schema / migration の意味変更
- 生成設定（`GENERATED by ...` のファイルは source を編集し再生成する。生成物を直接編集しない）
- 依存ロックファイル（`Cargo.lock`/`package-lock.json`/`poetry.lock` 等）の大幅変更
- 正本 docs（`docs/01`–`08`、`docs/03_domain_model.md` のドメイン意味）

## 変更リスク（レイヤ別。該当しない層は削る）

| Layer | リスク | 方針 |
|---|---:|---|
| domain（ドメイン意味 / 不変条件） | 高 | 意味変更 = owner |
| usecase / application | 中 | Goal/Done に紐づく時のみ |
| ports / adapters | 低〜中 | 主な編集領域 |
| composition root / 配線 | 中〜高 | env 切替境界、慎重に |
| infra / IaC / secret | 高 | owner approval（hook で停止） |
| CI/CD | 高 | owner approval（hook で停止） |
| tests | 低〜中 | 実装に合わせて弱めない |
| docs | 低 | 正本 docs は慎重（ADR 相当） |

## ルール

- タスクの Allowed Paths 内に留まる。保護パスに届く変更は Heavy / owner-approval の合図で、「ついで修正」ではない。
- 「ついでに共通化/改名/リファクタ」は task が明示しない限り scope creep（`control-change`）。
- メニューを削った/足した理由は `docs/decisions/decision-log.md` に残す。
